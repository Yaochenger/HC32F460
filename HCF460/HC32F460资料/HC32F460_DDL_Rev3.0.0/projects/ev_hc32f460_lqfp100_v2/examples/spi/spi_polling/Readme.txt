平台说明
================================================================================
GCC工程，由Eclipse IDE外挂GNU-ARM Toolchain，再结合pyOCD GDB Server实现工程的编译、
链接和调试。在用Eclipse导入工程后，请将xxxx_PyOCDDebug中pyocd-gdbserver和SVD文件
设置为正确的路径；请将xxxx_PyOCDDownload中pyocd设置为正确的路径。注意，这些路径不
能包含非英文字符。

================================================================================
功能描述
================================================================================
本样例展示SPI主机和从机之间通过查询方式进行数据收发功能。

说明：
本样例配置通过宏开关选择主机或从机模式，将主机代码下载到一块板子，从机代码下载到
另一块板子，通过排线将两块板子的SPI接口相连接，主机按下按键K2，触发一次数据收发，
主机和从机对收到的数据和发送的数据进行比较，相同的LED_BLUE亮，不同则LED_RED亮。

================================================================================
使用步骤
================================================================================
1）使用杜邦线连接2块板子的通信引脚，引脚连接对应关系如下：
    目标板1                     目标板2
    J1 SCK  <-----------------> J1 SCK
    J1 MOSI <-----------------> J1 MOSI
    J1 MISO <-----------------> J1 MISO
    J1 NSS  <-----------------> J1 NSS
    J1 GND  <-----------------> J1 GND
2）打开工程，设置宏“EXAMPLE_SPI_MASTER_SLAVE”的值为“SPI_MASTER”，并重新编译工程
后下载到目标板1中；
3）修改宏“EXAMPLE_SPI_MASTER_SLAVE”的值为“SPI_SLAVE”，并重新编译工程后下载到目标
板2中；
4）复位运行目标板1的主机代码；
5）复位运行目标板2的从机代码；
6）按下目标板1的按键K2，触发主机和从机之间全双工的数据收发，并在传输完成后执行比
较功能；
7）收发数据相同则LED_BLUE亮，不同则LED_RED亮；
8）可重复按下目标板1的按键K2，执行6、7的操作。
