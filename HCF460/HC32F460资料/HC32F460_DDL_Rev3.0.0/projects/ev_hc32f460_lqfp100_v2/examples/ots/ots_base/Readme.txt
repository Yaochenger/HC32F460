平台说明
================================================================================
GCC工程，由Eclipse IDE外挂GNU-ARM Toolchain，再结合pyOCD GDB Server实现工程的编译、
链接和调试。在用Eclipse导入工程后，请将xxxx_PyOCDDebug中pyocd-gdbserver和SVD文件
设置为正确的路径；请将xxxx_PyOCDDownload中pyocd设置为正确的路径。注意，这些路径不
能包含非英文字符。

================================================================================
功能描述
================================================================================
本例程实现了用OTS测量MCU内部温度。

说明：
1. OTS可选用内置的默认参数，也可使用通过定标实验获得的参数，通过‘OTS_USE_DEFAULT_PARA’
   指定。‘OTS_USE_DEFAULT_PARA’为非0时，选用内置的默认参数，不需要定标实验；
   ‘OTS_USE_DEFAULT_PARA’为0时，必须先通过定标实验获得参数，才能使用OTS进行测温，每颗
   MCU都必须做定标实验。
2. 将‘OTS_USE_TRIG’定义为非0，可使用外部触发源触发OTS工作。本例程采用timer0每秒触发一次OTS。
3. 将‘OTS_USE_INTERRUPT’定义为非0，可使用OTS中断。
4. 例程当前的配置为使用内置的默认参数，使用外部触发源触发OTS工作。

================================================================================
使用步骤
================================================================================
1）用 DAP 接口调试，可以直接通过串口调试助手查看程序运行情况；
2）打开工程，重新编译，启动调试或直接下载程序运行；
3）用设置断点的方式查看温度值，或用USB转串口结合串口调试助手软件，在上位机观察温度值，
   温度值每秒输出一次；
