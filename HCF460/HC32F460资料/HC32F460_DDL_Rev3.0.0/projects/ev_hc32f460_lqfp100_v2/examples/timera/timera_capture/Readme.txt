平台说明
================================================================================
GCC工程，由Eclipse IDE外挂GNU-ARM Toolchain，再结合pyOCD GDB Server实现工程的编译、
链接和调试。在用Eclipse导入工程后，请将xxxx_PyOCDDebug中pyocd-gdbserver和SVD文件
设置为正确的路径；请将xxxx_PyOCDDownload中pyocd设置为正确的路径。注意，这些路径不
能包含非英文字符。

================================================================================
功能描述
================================================================================
本例程用TimerA单元1的通道4展示了TimerA的捕获功能。通过配置，可实现对TIMERA的PWM引脚（PA11）
和TRIG引脚（PA12）的边沿以及指定事件（EVT_SRC_PORT_EIRQ12，按下K1产生）的捕获。

================================================================================
使用步骤
================================================================================
1）用 DAP 接口调试，可以直接通过串口调试助手查看程序运行情况；
2）打开工程，重新编译，启动调试或直接下载程序运行；
3）例程默认捕获 PA11（PWM 引脚） 的上升沿。将 PA11 与 PB1（连着按键 K10）短接。
   按下 K10，可看到串口打印捕获到的计数器值。
4）将宏 TMRA_CAPT_COND 定义为 TMRA_CAPT_COND_TRIG_RISING，捕获 PA12（TRIG 引脚）的
   上升沿。将 PA12 与 PB1（连着按键 K10）短接。
   重新编译程序，启动调试或直接下载程序运行，按下 K10，可看到串口打印捕获到的计数器值。
5）将宏 TMRA_CAPT_COND 定义为 TMRA_CAPT_COND_EVT，捕获事件，例程中设置的用来捕获的事
   件是 EVT_SRC_PORT_EIRQ12（按下按键 K1 产生）。重新编译程序，启动调试或直接下载程序
   运行，按下 K1，可看到串口打印捕获到的计数器值。由于按键扫描的原理，按键按下一次，可能
   产生多次事件 EVT_SRC_PORT_EIRQ12，从而导致捕获产生多次，串口打印多次。
