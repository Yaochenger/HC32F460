平台说明
================================================================================
GCC工程，由Eclipse IDE外挂GNU-ARM Toolchain，再结合pyOCD GDB Server实现工程的编译、
链接和调试。在用Eclipse导入工程后，请将xxxx_PyOCDDebug中pyocd-gdbserver和SVD文件
设置为正确的路径；请将xxxx_PyOCDDownload中pyocd设置为正确的路径。注意，这些路径不
能包含非英文字符。

================================================================================
功能描述
================================================================================
本例程用TimerA单元1的通道6和通道5为例，展示了TimerA的比较值缓存功能。例程中，通道6的
比较值寄存器作为通道5的比较值寄存器的缓存，当指定条件产生时，通道6的比较值寄存器值会被
传送到通道5的比较值寄存器中。

说明：
1. 例程中使用了TimerA中断，只是为了查看比较值缓存情况。
2. 例程默认使用三角波计数模式，在计数到波峰和波谷是都会进行比较值传送，将通道6的比较值
   传送至通道5的比较值寄存器中。
3. 例程在上溢中断函数和下溢中断函数中输出比较值缓存情况。

================================================================================
使用步骤
================================================================================
1）用 DAP 接口调试，可以直接通过串口调试助手查看程序运行情况；
2）打开工程，重新编译，启动调试或直接下载程序运行；
3）通过串口调试助手，可看到在上溢中断函数和下溢中断函数中都会输出比较值缓存情况：
   串口打印类似如下信息：
   UDF Irq callback set compare value buffer: 4106
   OVF Irq callback get current compare value: 4106
   OVF Irq callback set compare value buffer: 4107
   UDF Irq callback get current compare value: 4107
   UDF Irq callback set compare value buffer: 4108
   OVF Irq callback get current compare value: 4108
   ......
   说明：
   在 UDF 中设置缓存值为 4106，然后 OVF 产生，在 OVF 中读取到比较值 4106，表示缓存值传送成功
   在 OVF 中设置缓存值为 4107，然后 UDF 产生，在 UDF 中读取到比较值 4107，表示缓存值传送成功
   ......
4）修改宏 TMRA_MD 的定义为 TMRA_MD_SAWTOOTH（锯齿波模式，例程中，该模式下计数方向为向下），
   重新编译，启动调试或直接下载程序运行，可看到在下溢中断函数中输出比较值缓存情况。
