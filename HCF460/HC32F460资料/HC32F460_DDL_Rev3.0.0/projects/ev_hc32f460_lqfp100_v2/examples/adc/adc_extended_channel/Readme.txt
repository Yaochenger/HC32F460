平台说明
================================================================================
GCC工程，由Eclipse IDE外挂GNU-ARM Toolchain，再结合pyOCD GDB Server实现工程的编译、
链接和调试。在用Eclipse导入工程后，请将xxxx_PyOCDDebug中pyocd-gdbserver和SVD文件
设置为正确的路径；请将xxxx_PyOCDDownload中pyocd设置为正确的路径。注意，这些路径不
能包含非英文字符。

================================================================================
功能描述
================================================================================
本例程以ADC1的扩展通道（内部通道）为例，展示了如何用ADC获取内部参考电压和8bit-DAC（DAC1）
的输出电压。例程的默认使用ADC1的扩展通道获取8bit-DAC（DAC1）的输出电压（2.2V）。

例程功能说明：
1. 用‘ADC_INTERN_SRC_SEL’指定获取内部参考电压（约1.1V）或8bit-DAC输出电压；
2. ‘ADC_VREF’为引脚‘VREFH’的电压，即ADC的参考电压；
3. 用‘DAC_VAL’指定8bit-DAC的输出电压，例程中设置的输出电压为2.2V。

================================================================================
使用步骤
================================================================================
1）用 DAP 接口调试，可以直接通过串口调试助手查看程序运行情况；
2）打开工程，重新编译，启动调试或直接下载程序运行；
3）通过串口调试助手软件，可看到 ADC 获取到的电压值约为 2.2V（8bit-DAC输出电压）；
4）修改宏 ADC_INTERN_SRC_SEL 的定义为 ADC_INTERN_SRC_REF_VOL（获取内部参考电压），
   重新编译，启动调试或直接下载程序运行，可看到 ADC 获取到的电压值约为 1.1V

================================================================================
注意
================================================================================
ADC1对应的扩展通道为‘ADC1_EXT_CH’，ADC2对应的扩展通道为‘ADC2_EXT_CH’。
