平台说明
================================================================================
GCC工程，由Eclipse IDE外挂GNU-ARM Toolchain，再结合pyOCD GDB Server实现工程的编译、
链接和调试。在用Eclipse导入工程后，请将xxxx_PyOCDDebug中pyocd-gdbserver和SVD文件
设置为正确的路径；请将xxxx_PyOCDDownload中pyocd设置为正确的路径。注意，这些路径不
能包含非英文字符。

================================================================================
功能描述
================================================================================
本例程展示了如何配置和使用ADC的模拟看门狗（AWD），并将模拟看门狗的窗口比较结果通过串口
输出。

说明：
1. 每个ADC单元只有一个AWD窗口，可以选择多个通道作为AWD通道，这些通道可在序列A中使能或
   在序列B中使能，也可一部分在序列A，另一部分在序列B；
2. 例程中AWD工作在中断模式；
3. AWD有两个中断源：
   1）通道比较中断：任何一个通道满足AWD比较条件，该通道AWD标志位置位，并产生中断；
   2）序列比较中断：若序列A或序列B中有AWD通道，在序列扫描结束后，若其中有通道满足AWD比较
      条件，则产生中断；
4. 在应用中，可只使用其中一个中断源，也可以同时使用两个中断源（需做两次中断配置）。本例程
   只使用了其中一个中断源，通道比较中断；
5. 本例程中的AWD通道属于序列B，通道对应的引脚为PC0，PC0与电位器相连，旋转电位器可调节电压；
6. 序列A由Timer0每秒触发一次；
7. AWD置位条件：1.5V <= PC0的电压 <= 2.5V。

================================================================================
使用步骤
================================================================================
1）用 DAP 接口调试，可以直接通过串口调试助手查看程序运行情况；
2）打开工程，重新编译，启动调试或直接下载程序运行；
3）旋转电位器 R76 ADC，在上位机通过串口调试助手软件或通过断点查看 AWD 的窗口比较情况：
   串口会一直打印 AWD channel voltage is xxxx
   当 1.5V <= PC0的电压 <= 2.5V 时，串口打印 --->> AWD channel flag set, voltage is xxxx
