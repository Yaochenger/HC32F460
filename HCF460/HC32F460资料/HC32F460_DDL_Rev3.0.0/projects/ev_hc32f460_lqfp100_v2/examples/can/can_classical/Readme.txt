平台说明
================================================================================
GCC工程，由Eclipse IDE外挂GNU-ARM Toolchain，再结合pyOCD GDB Server实现工程的编译、
链接和调试。在用Eclipse导入工程后，请将xxxx_PyOCDDebug中pyocd-gdbserver和SVD文件
设置为正确的路径；请将xxxx_PyOCDDownload中pyocd设置为正确的路径。注意，这些路径不
能包含非英文字符。

================================================================================
功能描述
================================================================================
本例程实现了经典CAN的数据发送和接收，并将接收到的数据通过串口输出，可在上位机用串口调试
助手软件查看。

说明：
1. 波特率设置：
   1）系统时钟200MHz；
   2）CAN通信时钟为外部晶振频率，8MHz；
   3）CAN位时间分频为通信时钟的1分频，即8MHz；
   4）CAN通信波特率为1Mbps，位时间为8TQ，采样点为75%。
2. 接收筛选器设置：
   1）标准ID 0x701，ID掩码为0，只接收标准ID为0x701的帧；
   2）扩展ID 0x12131415，ID掩码为0x000000F0，可接收扩展ID为0x121314x5的帧；
   3）标准和扩展ID 0x1A1B1C1D，ID掩码为0x0000000F，可接收扩展ID为0x1A1B1C1x的帧和标准
      ID为0x41x的帧。
3. 发送设置：
   1）按下并释放按键K1 第一次，用PTB发送，ID为标准ID 0x601；
   2）按下并释放按键K1 第二次，用STB发送，ID为标准ID 0x602；
   3）按下并释放按键K1 第三次，用PTB和STB，ID均为扩展ID，
      分别为 0x1020301，0x1030402，0x1030503

测试环境
================================================================================
测试用板:
---------------------
EV_F460_LQ100_Rev2.0

辅助工具:
---------------------
支持CAN通信的设备（简称CAN设备）

辅助软件:
---------------------
串口调试助手软件（115200bps，8N1）

================================================================================
使用步骤
================================================================================
1）用 DAP 接口调试，可以直接通过串口调试助手查看程序运行情况；
2）例程中 CAN 通信波特率为 1Mbps，采样点为75%。将 CAN 设备做同样设置（见波特率设置）后，
   将 CANH 和 CANL 分别与开发板上 CANH 和 CANL 连接（J7，电位器 R75 旁）；
3）打开工程，重新编译，启动调试或直接下载程序运行；
4）按下并释放按键 K1 第一次，CAN 设备会收到一帧 ID 为标准 ID 0x601 数据帧；
   按下并释放按键 K1 第二次，CAN 设备会收到一帧 ID 为标准 ID 0x602 数据帧；
   按下并释放按键 K1 第三次，CAN 设备会收到三帧报文，ID 均为扩展 ID，分别为 0x1020301，
   0x1030402，0x1030503；
   再按下并释放按键 K1，循环。
5）CAN 设备向开发板发送数据帧，ID 为标准 ID 0x701，串口调试助手会打印收到的数据帧；
   CAN 设备向开发板发送数据帧，ID 为扩展 ID 0x12131415，串口调试助手会打印收到的数据帧；
   CAN 设备向开发板发送数据帧，ID 为扩展 ID 0x1A1B1C1D，串口调试助手会打印收到的数据帧；
   CAN 设备向开发板发送数据帧，ID 为标准 ID 0x41D，串口调试助手会打印收到的数据帧；

================================================================================
注意
================================================================================
CAN控制逻辑时钟（EXCLK）频率不能低于CAN通信时钟频率。
