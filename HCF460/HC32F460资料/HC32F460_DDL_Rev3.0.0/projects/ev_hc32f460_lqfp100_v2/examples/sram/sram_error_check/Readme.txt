平台说明
================================================================================
GCC工程，由Eclipse IDE外挂GNU-ARM Toolchain，再结合pyOCD GDB Server实现工程的编译、
链接和调试。在用Eclipse导入工程后，请将xxxx_PyOCDDebug中pyocd-gdbserver和SVD文件
设置为正确的路径；请将xxxx_PyOCDDownload中pyocd设置为正确的路径。注意，这些路径不
能包含非英文字符。

================================================================================
功能描述
================================================================================
本例程演示了如何制造SRAM奇偶校验错误和ECC校验错误，以及校验错误后操作（系统服务或产生NMI
中断）。

说明：
1. 奇偶校验错误。读取一个未初始化的SRAM地址，会产生奇偶校验错误。
2. ECC校验错误。如下几个步骤的操作会产生ECC校验错误：
   1）设置ECC校验模式为非无效模式；
   2）对设置了ECC校验模式的SRAM的地址addr行写和读；
   3）设置ECC模式为无效模式，并写一个与第二步写的值不同的值到addr；
   4）重新设置ECC模式为第一步设置的模式；
   5）读取addr，将会产生ECC校验错误

================================================================================
使用步骤
================================================================================
1）打开工程，重新编译，启动调试或直接下载程序运行；
2）若是直接下载运行程序，需在下载程序后对板子重新上电；
3）程序默认检查 SRAM 奇偶校验错误，若产生奇偶校验错误，则产生 NMI 中断。
   程序默认现象：蓝灯亮，红灯灭；按下按键 K1，蓝灯灭，红灯亮。
4）修改宏 SAMPLE_FUNC 的定义为 SAMPLE_FUNC_SRAM_ECC_CHECK，检查 ECC 错误。
   重新编译，启动调试或直接下载程序运行。
   程序默认现象：蓝灯亮，红灯灭；按下按键 K1，蓝灯灭，红灯亮。
