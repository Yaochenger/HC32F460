平台说明
================================================================================
GCC工程，由Eclipse IDE外挂GNU-ARM Toolchain，再结合pyOCD GDB Server实现工程的编译、
链接和调试。在用Eclipse导入工程后，请将xxxx_PyOCDDebug中pyocd-gdbserver和SVD文件
设置为正确的路径；请将xxxx_PyOCDDownload中pyocd设置为正确的路径。注意，这些路径不
能包含非英文字符。


功能描述
================================================================================
本样例为芯片掉电模式、唤醒切换。
通过按下按键k10使芯片进入掉电模式；Wakeup timer用于唤醒MCU。

---------------------
串口助手

================================================================================
使用步骤
================================================================================
1）打开工程并重新编译；
2）设置串口助手115200bps@8N1，并打开PC端所识别到的COM口；
3）启动IDE的下载和调试功能，全速运行,蓝色LED闪烁1次；
4）按下按键k10，红色LED闪烁1次后，芯片随即进入掉电模式；
   此时，"MCU will entry power down mode..."会显示在串口终端；
5）Wakeup timer计数溢出后，芯片唤醒（复位）；
6）串口终端将打印出复位源；
   "Power down mode reset occurs."
7）可任意按下复位按键K11或k10，以观测串口终端打印信息来获取芯片状态。
8）按下复位按键K11，串口中断会打印出 “Pin reset occurs.”

================================================================================
注意
================================================================================
掉电模式3和掉电模式4不能用WKTM唤醒。
